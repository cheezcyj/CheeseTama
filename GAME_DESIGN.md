# CheeseTama: The Milkroom 게임 기획 요약

이 문서는 현재 Unity 프로토타입의 구현 방향, v2.0 아트 기준, v2.1 기술/폴더/작업 규칙을 공개 저장소에 둘 수 있는 범위로 정리합니다. 전체 기획 원문은 별도 내부 기획안을 기준으로 하며, 공개 README에는 해금 전 후반 콘텐츠와 로컬 작업 경로를 노출하지 않습니다.

## 현재 방향

`치즈타마: 밀크룸`은 밀크룸에 머물며 작은 치즈 생명체를 돌보는 PC 우선 감성 캐주얼 육성 게임입니다. 플레이어는 하루 1-3회 정도 접속해 10-30분가량 머물고, 상태를 관리하며 성장 기록과 방 꾸미기 요소를 천천히 확장합니다.

첫인상은 단순하고 포근해야 합니다. 귀여운 치즈 알이 말랑한 노란 치즈 생명체로 자라나는 흐름을 중심에 두고, 해금 전의 후반 콘텐츠는 UI, README, 도감, 홍보 문구에 노출하지 않습니다.

## 핵심 반복 구조

1. 밀크룸에 들어갑니다.
2. CheeseTama의 상태, 이벤트 메시지, 상태 메시지를 확인합니다.
3. 우유, 요리, 간식, 놀이, 청소, 수면 행동을 사용합니다.
4. CheeseTama의 표정, 통통 튀는 반응, 이벤트 단서를 확인합니다.
5. 방에 머물며 가벼운 체류 보상을 받습니다.
6. 발견한 기록을 도감에서 확인합니다.
7. 주요 상호작용 후 자동 저장하고, 설정에서 수동 저장 도구를 제공합니다.

## v2.0 레퍼런스

캐릭터 레퍼런스:

- 파일: 내부 캐릭터 디자인 포트폴리오 이미지
- 목적: CheeseTama의 기본 형태, 성장 단계, 표정, 기능 액션, 재질, 색상, 크기 기준
- 핵심 키워드: 말랑함, 치즈 질감, 둥근 실루엣, 큰 광택 눈, 복숭아빛 볼, 작은 팔/발, 얕은 치즈 구멍

밀크룸 레퍼런스:

- 파일: 내부 밀크룸 배경 디자인 포트폴리오 이미지
- 목적: 방 구성, 기능 존, 시간대/상태 변화, 소품, 색상 팔레트, 조명 기준
- 핵심 키워드: 아늑함, 우유빛, 따뜻함, 목재 디오라마, 정면 고정 카메라, 기능 존 분리

레퍼런스 이미지는 2D 배경판으로 사용하지 않습니다. 실제 3D 캐릭터, 소품, 방 모델링과 배치의 가이드로 사용합니다.

## 캐릭터 방향

CheeseTama는 고정 카메라 디오라마 안의 풀 3D 스타일라이즈드 툰 캐릭터입니다. 부드러운 노란 치즈 푸딩 같은 몸, 큰 눈, 복숭아빛 볼, 작은 팔과 발, 얕은 치즈 구멍, 광택 하이라이트가 핵심입니다.

성장 단계:

| 단계 | 이름 | 기준 |
| --- | --- | --- |
| Lv.1 | 치즈타마 알 | 치즈 질감이 있는 알 형태, 둥근 실루엣 |
| Lv.10 | 부화 | 몸체는 완전한 CheeseTama이며 껍질은 표면/하단/외곽 조각으로만 표현 |
| Lv.15 | 말랑 치즈타마 | 작은 팔/발과 얼굴이 또렷해지는 기본형 |
| Lv.20 | 성장한 치즈타마 | 몸집과 구멍, 하이라이트가 조금 더 풍부해짐 |
| Lv.28 | 숙성 치즈타마 | 치즈 무늬와 실루엣이 안정적으로 성숙 |
| Lv.33 | 최종형 | 레퍼런스의 대표 완성형 비율과 디테일 |

표정 기준:

- 기본/행복
- 배부름
- 배고픔
- 졸림
- 슬픔
- 불편함/심통
- 반짝임

기능 액션 기준:

- 우유 마시기
- 놀이
- 낮잠
- 청소 반응
- 레벨업

최종 아트 에셋 전까지 절차적 오브젝트를 사용할 수 있지만, 단순 원형 덩어리처럼 보이면 안 됩니다. 커스텀 말랑 메시, 카툰 재질, 외곽선, 표정 파츠, 치즈 무늬, 광택, 그림자, 부드러운 반응 모션이 필요합니다.

## 밀크룸 방향

밀크룸은 정면 고정 카메라로 보는 3D 디오라마입니다. 화면 첫인상에서 CheeseTama와 방의 기능 구역이 함께 읽혀야 하며, 하단 UI가 캐릭터나 주요 소품을 가리지 않아야 합니다.

기능 존:

- 칠판
- 창가
- 우유 선반
- 냉장고
- 휴식 의자
- 중앙 러그
- 요리 테이블
- 장식 소품

대표 소품:

- 우유병
- 냉장고
- 블렌더
- 별 조명
- 휴식 의자
- 러그
- 식물
- 수납장/선반

분위기 변형:

- 기본 밀크룸 낮
- 기본 밀크룸 밤
- 별빛 밀크룸
- 따뜻한 오후 밀크룸
- 겨울 밀크룸
- 빈티지 밀크룸

상태/시간대 변화:

- 아침
- 오후
- 밤
- 수면 모드
- 청소 전
- 청소 후

색상과 재질은 크림, 버터 옐로, 우유빛 흰색, 따뜻한 목재, 부드러운 파스텔, 은은한 푸른 밤색을 중심으로 합니다. 플레이 모드의 테마 처리는 GLB 소품의 PBR 재질 색상과 밝기를 덮어쓰지 않아야 합니다.

## 밀크룸 화면 규칙

직접 돌봄 행동과 시스템 메뉴를 명확히 분리합니다.

상단 상태 바:

- CheeseTama 이름
- 레벨과 진행도
- 세션 시간과 오늘 체류 시간
- 초기 재화 수량
- 상단 메뉴: 도감, 꾸미기, 설정

중앙 화면:

- CheeseTama가 중앙에서 잘 보여야 합니다.
- 일반 플레이 중 패널이 캐릭터를 가리지 않아야 합니다.
- 방의 바닥, 러그, 핵심 소품이 하단 UI에 과하게 가려지지 않아야 합니다.

메시지 영역:

- 이벤트 메시지 바는 상태 메시지 바 바로 위에 별도로 둡니다.
- 이벤트가 없으면 이벤트 메시지 바는 숨깁니다.
- 일반 돌봄 결과와 시스템 메시지는 상태 메시지 바에 표시합니다.

상태 수치 바:

- 포만감
- 기분
- 청결
- 졸림
- 건강

밀크룸 기록창:

- 형태, 상태, 성장, 우유, 루틴, 저장 기록을 섹션별로 분리합니다.
- 각 본문 라인의 제목은 굵게 표시하고 값은 같은 줄에서 이어 표시합니다.
- 섹션 간 상하 여백은 상태 수치 바의 행간과 비슷한 밀도로 유지합니다.

하단 행동 바:

```text
우유 / 요리 / 간식 / 놀이 / 청소 / 수면
```

도감, 꾸미기, 저장, 불러오기, 초기화, 개발자 기능, 1시간 경과 버튼은 하단 행동 바에 넣지 않습니다. 숫자 1-6은 여섯 직접 돌봄 행동 단축키로 사용합니다.

## 카메라와 화면 비율

- 기준 화면은 16:9입니다.
- 초광각, 21:9, 19:6, 세로형 화면에서도 캐릭터 발과 주요 소품이 하단 UI 위에 보여야 합니다.
- `MilkroomCameraFramer`는 실제 표시 중인 하단 UI 바를 기준으로 여백을 계산합니다.
- 숨겨진 이벤트 메시지 바는 카메라 리프트 계산에 포함하지 않습니다.
- 빌더는 밀크룸 카메라에만 프레이머를 붙이고, 디버그 카메라에는 붙이지 않습니다.
- 개발자 화면에서 방이 과하게 위로 올라가면 먼저 하단 UI 활성 상태, 참조 Canvas, 프레이머가 붙은 카메라를 확인합니다.

## 재질과 조명 규칙

- 절차적 밀크룸 오브젝트는 테마 팔레트로 색을 바꿀 수 있습니다.
- GLB로 임포트한 캐릭터/소품은 원본 PBR 재질과 텍스처를 우선 보존합니다.
- 플레이 모드에서 소품이 과하게 어두워지거나 색이 바뀌면 `MilkroomThemeController.ShouldPreserveImportedRenderer`를 먼저 확인합니다.
- 냉장고, 우유 선반, 안락의자 등 생성형 GLB 소품은 테마 팔레트 대상에서 제외합니다.
- Scene View가 아니라 Play Mode의 Game View 기준으로 최종 밝기와 구도를 판단합니다.

## 설정

도감과 꾸미기는 상단 메뉴에서 열립니다. 설정도 상단 메뉴에서 열리며 현재는 데이터 관리 기능을 포함합니다.

- 저장
- 불러오기
- 초기화

초기화는 반드시 `RESET`을 정확히 입력해야 실행 버튼이 활성화됩니다.

## 개발자 패널

개발자 패널은 에디터와 개발 빌드에서만 사용합니다. 상태 수치를 가리지 않도록 배치하며, 현재 기능은 다음과 같습니다.

- 1시간 경과
- 개발자 씬 이동

개발자/테스트 도구는 일반 플레이용 하단 행동 바에 넣지 않습니다.

## 도감과 숨김 규칙

- 도감은 상단 메뉴에서 엽니다.
- 발견한 우유, 성장, 진화, 이벤트 기록만 보여 줍니다.
- 숨겨진 후반 기록은 해금 전까지 슬롯, 이름, 희귀도, 분류, 전체 개수, 해금 조건을 노출하지 않습니다.
- 해금 후에도 정확한 성장 조건은 사용자 화면에 직접 노출하지 않습니다.
- 내부 데이터에 존재하는 후반 루트와 특수 타입은 공개 README, 첫 화면, 홍보 문구에 쓰지 않습니다.

## 유니티 작업 메모

- 로컬 작업 경로와 개인 PC 경로는 공개 문서에 쓰지 않습니다.
- 코드 변경 후 Unity에서 새로고침합니다.
- `CheeseTama > 시작 씬 빌드`를 실행해 시작 씬 오브젝트를 재생성합니다.
- `Milkroom` 씬을 열고 재생 버튼을 누릅니다.
- 의미 있는 변경 후 공개 가능한 변경분만 Git에 반영하고 커밋/푸시합니다.

3. 하단 바가 우유, 요리, 간식, 놀이, 청소, 수면만 표시하는지 확인합니다.
4. 각 돌봄 행동을 눌러 수치, 이벤트 메시지, 상태 메시지, 캐릭터 반응이 갱신되는지 확인합니다.
5. 이벤트가 없을 때 이벤트 메시지 바가 빈 박스로 남지 않는지 확인합니다.
6. 개발자 패널이 상태 바를 가리지 않는지 확인합니다.
7. `1시간 경과`로 시간 진행이 반영되는지 확인합니다.

비주얼:

1. CheeseTama가 레퍼런스처럼 둥글고 말랑한 치즈 생명체로 보이는지 확인합니다.
2. 눈, 볼, 입, 치즈 구멍, 작은 팔/발, 하이라이트가 카메라 정면에서 읽히는지 확인합니다.
3. 냉장고, 우유 선반, 안락의자 등 GLB 소품 색상과 밝기가 플레이 모드에서 깨지지 않는지 확인합니다.
4. 방이 개발자 화면에서 과하게 위로 올라가 있지 않은지 확인합니다.
5. 16:9보다 짧은 화면 비율에서도 캐릭터 발과 러그가 하단 UI 위에 남는지 확인합니다.

설정:

1. 설정을 엽니다.
2. 저장, 불러오기, 초기화 버튼이 글자와 겹치지 않는지 확인합니다.
3. 저장과 불러오기가 상태 메시지를 갱신하는지 확인합니다.
4. 초기화 창을 열고 `RESET` 입력 전에는 버튼이 비활성인지 확인합니다.

도감:

1. 우유와 돌봄 행동을 수행합니다.
2. 상단 메뉴에서 도감을 엽니다.
3. 발견한 기록이 표시되는지 확인합니다.
4. 숨겨진 기록이 해금 전 노출되지 않는지 확인합니다.

## 빌드 검증

```powershell
dotnet restore CheeseTama.csproj

```

## 구현 규칙

- 첫 화면은 포근하고 읽기 쉬우며 CheeseTama가 중심이어야 합니다.
- 직접 돌봄 행동과 시스템/개발자/이동 도구를 분리합니다.
- 숨겨진 후반 콘텐츠는 해금 전 완전히 숨깁니다.
- 레퍼런스 이미지는 3D 모델링/배치 가이드로 쓰고 평면 배경으로 붙이지 않습니다.
- 생성형 GLB 소품의 PBR 재질을 플레이 모드 테마가 덮어쓰지 않게 합니다.
- 원본 생성 결과는 `GeneratedAssets`에 보관하되 런타임 참조는 정리된 `Assets/...` 경로만 사용합니다.
- README는 공개용 문서로 유지하고, 숨김 콘텐츠의 조건/개수/이름은 쓰지 않습니다.
- 먹이 페널티는 영구 처벌처럼 느껴지지 않게 합니다.
- PC 마우스 우선 조작을 기준으로 하되, 향후 입력 확장을 막지 않습니다.
