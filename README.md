# CheeseTama: The Milkroom

`치즈타마: 밀크룸`은 우유로 자라는 작은 치즈 생명체를 돌보는 PC 우선 감성 캐주얼 육성 게임입니다. 플레이어는 포근한 밀크룸에 들어가 CheeseTama의 상태를 확인하고, 우유/요리/간식/놀이/청소/수면 행동으로 성장과 기록을 쌓습니다.

현재 저장소는 최종 게임이 아니라, Unity에서 핵심 반복 구조와 3D 비주얼 방향을 검증하는 개발 버전입니다.

## 현재 상태

- Unity 6 `6000.0.78f1` 기반 프로젝트
- MIT 라이선스
- URP, uGUI, glTFast 기반 3D/화면 프로토타입
- 부트, 밀크룸, 도감, 개발자 씬 구성
- 로컬 저장/불러오기/초기화
- 밀크룸 체류 시간, 일일 루틴, 초기 재화, 도감 기록 프로토타입
- 우유, 요리, 간식, 놀이, 청소, 수면 행동 프로토타입
- 3D 스타일라이즈드 툰 렌더링 방향의 CheeseTama 캐릭터
- 레퍼런스 이미지 기반 밀크룸 3D 디오라마와 주요 소품 프리팹 배치
- 생성형 GLB 캐릭터/소품 프리팹 임포트와 PBR 재질 유지 흐름
- 깔끔한 밀크룸 방 쉘, 냉장고, 우유병 선반, 원목 의자, 창문, 러그, 수납장탁자, 칠판 중심의 현재 씬 구성
- 상태 메시지와 이벤트 메시지 분리
- 상태 수치창과 밀크룸 기록창의 좌우 분리, 기록 제목 강조와 섹션 여백 기반 UI 가독성 보정
- 여러 화면 비율에서 하단 UI가 캐릭터와 방을 가리지 않도록 카메라 프레이밍 보정
- 설정 데이터 관리, 도감/꾸미기 오버레이, 개발자 패널

## 개발 기준

- 주요 게임 코드는 `Assets/_Project/Scripts` 아래에서 도메인별로 관리합니다.
- 생성형 캐릭터 에셋은 `Assets/Characters`, 밀크룸 소품은 `Assets/Environments/Milkroom/Props`에 정리합니다.
- 밀크룸 소품은 GLB 임포트 프리팹과 에디터 빌더가 생성한 프리미티브 기반 프리팹을 같은 폴더에서 관리합니다.
- `GeneratedAssets`와 `Assets/AI Toolkit/Temp`는 생성 중간 산출물 보관용이며 런타임 기준 경로로 쓰지 않습니다.
- 정리된 소품 프리팹의 재질은 플레이 모드 테마가 덮어쓰지 않게 유지합니다.
- UI 패널과 버튼의 둥근 배경은 Unity 내장 PSD 리소스에 의존하지 않고 런타임 생성 스프라이트를 사용합니다.
- 내부 기획안과 로컬 작업 경로는 공개 README에 기록하지 않습니다.

## 비주얼 기준

캐릭터는 말랑한 노란 치즈 생명체를 기준으로 합니다. 둥근 몸체, 큰 광택 눈, 작은 팔과 발, 복숭아빛 볼, 치즈 구멍, 부드러운 하이라이트가 핵심입니다.

밀크룸은 정면 고정 카메라의 아늑한 3D 디오라마입니다. 현재 구현은 과한 장식을 줄인 깔끔한 벽과 바닥, 창문, 러그, 칠판, 수납장탁자, 냉장고, 우유병 선반, 원목 의자, CheeseTama가 한눈에 읽히는 구성을 우선합니다.

캐릭터/밀크룸 포트폴리오 이미지는 배경판으로 붙이지 않고, 실제 3D 모델링과 배치의 레퍼런스로 사용합니다.

## 실행 방법

1. Unity Hub에서 이 저장소 루트 폴더를 엽니다.
2. Unity 상단 메뉴에서 새로고침을 실행합니다.
3. 콘솔에 빨간 컴파일 오류가 없는지 확인합니다.
4. Unity 상단 메뉴에서 `CheeseTama > 시작 씬 빌드`를 실행합니다.
5. `Milkroom` 씬을 열고 재생 버튼을 누릅니다.

## 빠른 확인

1. 밀크룸 씬에서 재생합니다.
2. 하단의 `우유`, `요리`, `간식`, `놀이`, `청소`, `수면` 버튼을 눌러 봅니다.
3. 왼쪽 상태 수치, 오른쪽 밀크룸 기록, 이벤트 메시지, 상태 메시지, CheeseTama 반응이 갱신되는지 확인합니다.
4. 냉장고, 우유병 선반, 원목 의자의 색상과 밝기가 개발자 화면과 플레이 화면에서 크게 달라지지 않는지 확인합니다.
5. 상단의 `도감`, `꾸미기`, `설정` 메뉴를 확인합니다.
6. 16:9보다 짧은 화면 비율에서도 캐릭터와 밀크룸이 하단 UI 위에 보이는지 확인합니다.
7. 개발자 패널에서 `1시간 경과` 버튼을 눌러 시간 진행을 확인합니다.

## 빌드 검증

Unity가 프로젝트 파일을 만든 뒤에는 아래 명령으로 C# 컴파일을 확인할 수 있습니다.

```powershell
dotnet restore CheeseTama.csproj
dotnet build CheeseTama.csproj --no-restore
```
